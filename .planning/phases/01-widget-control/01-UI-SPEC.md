- props-grid-2 gap stays at 6px (established pattern — do not change)

## Component Inventory

Phase 1 adds new HTML+CSS to existing patterns. No new component primitives needed.

### New fields added to `#widget-props` in `studio.html`

#### Basic section additions (inside existing `<details class="props-section" open>` Basic block)

```
Subtitle / Description
  <input id="prop-subtitle" type="text" placeholder="e.g. Last 24h · GCP East">
  — renders below title on TV display as smaller secondary label
  — placed immediately after prop-title in DOM

Value Format
  <input id="prop-format" type="text" placeholder="${value}K or %.2f ms">
  — placed after prop-subtitle
  — hint text below input: "Tokens: ${value} ${unit}"
  — 10px IBM Plex Mono, color: var(--t3)
```

#### Labels section (new `<details class="props-section" id="labels-section">`)

Visibility: shown only for widget types with axis/legend fields. Hidden for: big-number, stat-card, gauge, gauge-row, sparkline, progress-bar, alert-list, service-heatmap, security-scorecard, usa-map, usa-map-gl.

Shown for: bar-chart, line-chart, stacked-bar-chart, donut-ring, sankey, heatmap, treemap, pipeline-flow, multi-metric-card, status-grid, table.

```
X Axis Label    <input id="prop-x-label" type="text" placeholder="e.g. Time">
Y Axis Label    <input id="prop-y-label" type="text" placeholder="e.g. Requests/s">
Legend / Stage Labels  <input id="prop-legend" type="text" placeholder="comma-separated, e.g. East,West,Central">
```

Fields shown per type:
- bar-chart, line-chart, stacked-bar-chart: X Axis Label + Y Axis Label
- donut-ring, pie: Legend / Stage Labels only
- pipeline-flow: Legend / Stage Labels (stage names)
- status-grid: Legend / Stage Labels (cell labels)
- multi-metric-card, table, sankey, heatmap, treemap: Legend / Stage Labels only

#### Position section — make inputs writable

Remove `readonly` attribute from prop-col, prop-row, prop-colspan, prop-rowspan.
Update hint text from "Drag to reposition · resize handles for col/row spans · or edit below" to "Enter values or drag widget on canvas — both stay in sync".
No visual style changes to the inputs themselves (opacity: 0.5 / cursor: default rules removed from non-readonly state).

#### Type switcher behavior

The `<select id="prop-type">` already exists. Phase 1 adds `(beta)` suffix to type option labels for unimplemented/partial renderers.

Unimplemented types get `(beta)` suffix in option text:
- sankey (beta)
- heatmap (beta)
- treemap (beta)
- table (beta)
- multi-metric-card (beta)
- stacked-bar-chart (beta)

All other types remain without suffix.

#### Data-shape mismatch warning (inline, not toast)

Shown directly below `#data-summary` when a type change occurs and no auto-matched query is found.

```html
<div id="type-mismatch-warning" class="type-mismatch-warning" style="display:none">
  <span class="warning-icon">⚠</span>
  <span class="warning-text">Data shape may not match this widget type. Verify or reassign the query.</span>
</div>
```

Style: background rgba(251,191,36,0.08), border 1px solid rgba(251,191,36,0.3), border-radius 4px, padding 6px 8px, font-size 11px, color var(--amber), font-family var(--font-display), letter-spacing 0.5px, display flex, align-items center, gap 6px.

#### Unimplemented widget placeholder on canvas

When a widget type has no working renderer or renders empty, the canvas cell shows:

```
Grey card fill: var(--bg-card-alt)
Centered content:
  — Type name in 11px Space Grotesk, color var(--t3), letter-spacing 1.5px, text-transform uppercase
  — Below: small badge "(beta)" — 9px, background rgba(251,191,36,0.12), border 1px solid rgba(251,191,36,0.3), color var(--amber), border-radius 3px, padding 1px 6px, font-family var(--font-mono)
```

No separate CSS class needed — the existing `.canvas-placeholder` pattern is the model; a new `.widget-placeholder` class is added following the same structure.

## Interaction Contracts

### Position input writability (WDGT-02)

- User edits prop-col, prop-row, prop-colspan, or prop-rowspan
- On `input` event: call existing `markDirty()` + trigger canvas re-render via `bind()` pattern
- Collision handling: if target slot is occupied, snap to nearest open adjacent slot (not reject)
  - Snap direction priority: right → down → left → up
  - Canvas drag state (`studio-canvas.js`) updates synchronously so inputs and drag state stay in sync
- Visual feedback during snap: no separate animation — the canvas re-renders and inputs update to resolved position

### Type switching (WDGT-03)

- User changes `prop-type` select
- Immediate canvas re-render (existing `bind()` → `renderCanvas()` already handles this)
- Auto-match attempt: search saved queries for same metric type compatible with new widget type
  - If found: assign silently, update `#data-summary`
  - If not found: preserve existing queryId, show `#type-mismatch-warning`
- Section visibility update: hide/show labels-section, map-config-section, mgl-config-section, display-section per new type
- Config preservation rule:
  - Keep: unit, format, min, max, warn, crit across compatible type switches
  - Clear: map config fields when leaving usa-map or usa-map-gl
  - Clear: mgl config fields when leaving usa-map-gl
  - Clear: axis/legend fields when switching to a type that doesn't use them

### Save flow (WDGT-01, WDGT-02, WDGT-03)

- No auto-save (decision locked in CONTEXT.md)
- Dirty state shown in `#dirty-indicator` (existing: amber text "● UNSAVED")
- Save button: existing `.studio-btn.primary` "Save" — no label change
- On save success: green toast "Dashboard saved" (existing toast pattern)
- On save error: red toast "Save failed — check connection and retry"
- No navigate-away warning, no Discard button

### Focus and keyboard

- All new inputs follow existing tab order (DOM order within the props panel)
- No custom keyboard shortcuts for Phase 1 inputs
- Inputs gain pink focus ring on :focus (existing: border-color var(--mh-pink), box-shadow 0 0 0 2px rgba(253,164,212,0.1))

## Section Priority Order

Properties panel section render order (top to bottom):

1. Basic — title, subtitle, type (type selector includes beta suffixes)
2. Data — source, query, data-summary, mismatch warning
3. Position — col, row, colspan, rowspan (now writable)
4. Display — unit, format, min, max, warn, crit
5. Labels — x-axis, y-axis, legend (shown only for applicable types)
6. Map Config — (existing, shown only for usa-map)
7. Map GL Config — (existing, shown only for usa-map-gl)
8. Danger zone — Delete Widget button (existing)

Source: CONTEXT.md `<specifics>` section.
