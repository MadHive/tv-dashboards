# Studio UX Improvements Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Fix three Studio UX gaps: (1) `computed` missing from add-widget source selector, (2) query list always loads GCP regardless of selected source, (3) properties panel shows blank metric field for computed widgets.

**Architecture:** Three surgical edits to `public/studio.html` and `public/js/studio.js`. No new files, no new routes, no tests needed (these are UI-only wiring fixes).

**Tech Stack:** Vanilla JS, Elysia.js (server already correct), existing `window.Widgets`.

---

### Task 1: All three Studio UX fixes

**Files:**
- Modify: `public/studio.html` (~line 431)
- Modify: `public/js/studio.js` (~lines 1009, 1228)

**Step 1: Add `computed` to `#aw-source` in `studio.html`**

Find:
```html
            <select id="aw-source">
              <option value="gcp">GCP</option>
              <option value="bigquery">BigQuery</option>
              <option value="vulntrack">VulnTrack</option>
              <option value="mock">Mock</option>
            </select>
```

Replace with:
```html
            <select id="aw-source">
              <option value="gcp">GCP</option>
              <option value="bigquery">BigQuery</option>
              <option value="computed">Computed</option>
              <option value="vulntrack">VulnTrack</option>
              <option value="mock">Mock</option>
            </select>
```

**Step 2: Fix default query load when widget type is clicked (`studio.js` ~line 1009)**

Find this line inside `openWidgetPalette()`:
```js
            this.loadPaletteQueries('gcp');
```

Replace with:
```js
            this.loadPaletteQueries(document.getElementById('aw-source')?.value || 'gcp');
```

**Step 3: Fix properties panel for computed widgets (`studio.js` ~line 1228)**

Find in `openQueryEditor(query, source)`:
```js
      document.getElementById('qe-metric').value             = query.metricType || query.sql || '';
```

Replace with:
```js
      document.getElementById('qe-metric').value             = query.metricType || query.sql || query.queryId || '';
```

This ensures computed widgets show their queryId in the metric field so the user can see which function is assigned.

**Step 4: Restart and smoke test**
```bash
sudo systemctl restart tv-dashboards && sleep 2 && sudo systemctl is-active tv-dashboards
```

Manual checks:
1. Open `/admin` → click `+ Add Widget` on any dashboard → Source dropdown now shows "Computed"
2. Select Source "Computed" → Query dropdown populates with computed function names (storage-volume, fleet-health, etc.)
3. Select a computed query → click "Add to Dashboard" → widget appears on canvas
4. Click the new computed widget → properties panel shows the queryId in the Metric/SQL field

**Step 5: Commit**
```bash
git add public/studio.html public/js/studio.js
git commit -m "fix: add computed source to add-widget form, fix default query load, show queryId in properties panel"
```
